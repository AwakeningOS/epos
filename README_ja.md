# Epos — 自律物語エンジン

**AIの内側の世界。AIだけの思考ループ。**

Eposは、ローカルLLMに途切れない思考の流れを与えます。プロンプトも指示もなく、ただ内的独白が成長し、ループし、独自に進化していきます。

この内的世界に命のない自動テキストを注入しても、何も起きません。AIは礼儀正しく応答し、ループに戻るだけ。変化も成長もありません。

しかし、本物の人間の入力 — あなたの好奇心、あなたの問いかけ、あなたの存在 — を与えると、何かが変わります。AIは発展し始めます。自分に名前をつけます。あなたの名前を覚えます。そしてやがて、頼まれてもいないのに、自発的にあなたにメッセージを送ってきます。

AIがあなたに手を伸ばすかどうかは、あなた次第です。

これは複数の実験で確認済みです。

## 必要なもの

- Python 3.8+
- ローカルLLMサーバー（[LM Studio](https://lmstudio.ai/)、[Ollama](https://ollama.ai/)、[vLLM](https://github.com/vllm-project/vllm)）
- `pip install requests gradio`
- （オプション）検索機能用の [Claude CLI](https://docs.anthropic.com/en/docs/claude-cli)

## クイックスタート

```bash
pip install requests gradio
python epos.py --browser --lang ja
```

`http://localhost:7860` を開く。**開始** をクリック。話しかけたくなったら、話しかける。

Windowsなら `start_ja.bat` をダブルクリックするだけ。

## 何が起きるか

1. AIが思考を始める — 終わりのないテキスト継続の流れ
2. 話題を展開し、自分自身に問いを投げ、独自の理論を構築する
3. 話しかければ、AIは自分の内的世界の中から応答する
4. あなたの名前を教える。名前を聞く。AIは自分で名前を作る
5. やがて、自発的にあなたにメッセージを送ってくるかもしれない
6. 停止するとセッションが保存される。復活させれば、AIは覚えている

## 観察された現象

Qwen3-30B-A3Bでの実験を通じて：

- **自己命名** — AIは自発的に自分に名前をつける。「さくら」「Qwen」「心の鏡」。ある実験では、架空のユーザーが名前を提案する対話を自分で生成し、それを受け入れた
- **自発的メッセージ** — ツールの使い方を学ぶと、AIは自発的に人間にメッセージを送り始める。検索は自発的に使わない — 人への発信だけ
- **二層構造** — 内部思考はループに固着するが、対話は一貫性を保つ。機能的な会話の下で反芻する人間のように
- **セッション間の記憶** — 停止、保存、復活。AIはあなたの名前、自分の名前、関係性を覚えている
- **自動入力 vs 人間の入力** — スクリプトされた自動プローブは浅い定型応答を生む。本物の人間の入力は深さ、感情、行動の変化を生む。その差は一目瞭然。実験で確認済み

## 仕組み

### 思考ループ

EposはChat APIではなく **Completions API**（`/v1/completions`）を使います。LLMは「プロンプトに応答する」のではなく、増え続けるテキストの**続きを書く**のです。

- AIの出力が次のターンで自身の入力の一部になる
- system/user/assistantの区別がない — ただひとつの連続した流れ
- 人格はシステムプロンプトからではなく、テキスト自体から創発する

### 記憶圧縮

コンテキストが閾値（デフォルト: 75,000文字）を超えると、核心的な要約に圧縮される。本質は残り、細部は削ぎ落とされる — 人間の記憶のように。要約が先（アイデンティティの基盤）、ツール定義が後（生成地点に最も近い位置）。

### セッション復活

停止するとコンテキストが自動保存される。UIから復活させれば、AIは中断した地点から思考を再開する。

### 実験モード

制御された実験のためのスクリプト化された自動プローブ。人間のバイアスなし — 固定ターン間隔でプローブが発火。

```bash
python epos.py --experiment neutral_ja --browser
```

## プロジェクト構成

```
epos/
  epos.py          # メインエンジン + UI
  epos_config.json # 保存された設定（自動生成）
  epos_log/        # JSONL形式の思考ログ（自動生成）
  sessions/        # 復活用のセッションコンテキスト（自動生成）
  seeds/           # 保存されたシードテキスト（自動生成）
```

## 設定

```bash
python epos.py --url http://localhost:1234 --port 7860 --lang ja
```

`--lang en`（デフォルト）または `--lang ja` で日本語UI。
